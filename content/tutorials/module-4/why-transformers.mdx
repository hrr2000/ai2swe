## The bottleneck: recurrence

**RNNs** (and LSTMs/GRUs) read tokens **left-to-right**, maintaining a hidden state vector `h_t`:

```javascript
let h = zeros(dim);
for (const token of sequence) {
  h = f(token, h); // step t waits on step t−1
}
```

That `for` loop is **serial**. A GPU with thousands of cores mostly **idles**, because timestep `t` cannot start until `t−1` finishes. Training on long corpora was painfully slow — and harder to scale across devices.

## The idea: attention instead of recurrence

The 2017 paper **“Attention Is All You Need”** proposed the **Transformer**: stack **self-attention** blocks so each position can **aggregate information from every other position** using weighted sums, with **no inherent sequential dependency inside a layer**.

In practice, implementations fuse operations into giant matrix multiplies — exactly what **tensor cores** excel at.

## Why this unlocked LLM scaling

Once depth and width became primarily **big batched GEMMs**, researchers could:

- Train on **far more data** in wall-clock time.
- Use **mixed precision**, **model parallelism**, and **pipeline parallelism** patterns borrowed from HPC.
- Build **deeper** stacks (dozens of layers) without vanishing-gradient pathologies that plagued deep RNNs.

The architectural bet matched the **hardware bet** — NVIDIA-scale GPUs — and the flywheel spun.

## What Transformers are *not*

They are not “more human-like memory.” They are a **parameterized, differentiable program** with strong **inductive biases** (multi-head mixing, position info, MLP sublayers). Their success is an engineering-science combo: **data + scale + attention + optimization tricks**.

<Callout type="insight">
**Self-attention** is the core novelty; everything else (FFNs, residuals, LayerNorm, positional schemes) is how we make deep stacks **trainable** and **stable**.
</Callout>

## Key takeaways

- RNNs were **sequential**; Transformers are **parallel-in-length** within layers.
- Hardware utilization flipped from “bad fit” to “great fit” for GPUs/TPUs.
- The revolution is **compute structure + scale**, not a single magical equation.
