## Specification

Input `X ∈ ℝ^{T×d}` — `T` tokens, `d` model dim. Learned weights `W_Q, W_K, W_V ∈ ℝ^{d×d}` (single-head for brevity).

```python
import torch
import torch.nn.functional as F

def attention(x, Wq, Wk, Wv, causal=True):
    # x: (T, d)
    T = x.shape[0]
    Q = x @ Wq
    K = x @ Wk
    V = x @ Wv
    scores = (Q @ K.T) / (Q.shape[-1] ** 0.5)  # (T, T)
    if causal:
        mask = torch.triu(torch.ones(T, T, device=x.device), diagonal=1).bool()
        scores = scores.masked_fill(mask, float("-inf"))
    attn = F.softmax(scores, dim=-1)
    return attn @ V  # (T, d)
```

## Exercises

1. **Numerical stability** — replace `-inf` mask with large negative for float16.
2. **Dropout** — `F.dropout(attn, p=0.1, training=True)` on weights row-wise.
3. **Multi-head** — reshape `d` → `h × (d/h)` and run `h` attentions in parallel (`einsum`).

<Callout type="warning">
Production uses **flash attention** kernels — different memory pattern. Your scratch version teaches **correctness**, not **speed**.
</Callout>

## Key takeaways

- Shapes `T×T` attention matrix make **quadratic** cost obvious.
- **Causal mask** is one `triu` boolean tensor.
- Implement once, then read **FlashAttention** papers with confidence.
