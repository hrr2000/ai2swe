## The issue

Attention treats input tokens as a **set** unless you tell it otherwise — reorder words, you get the same pairwise dot products (before masking). **Position information** must be added or multiplied in.

## Classic: sinusoidal (Transformer paper)

Add fixed sine/cosine curves of different frequencies to embedding vectors:

```text
PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

Smooth, extrapolates modestly beyond train length (in theory), no extra learned params.

## Learned absolute positions

Learn an embedding table `E[pos]` like token embeddings — simple, but **length extrapolation** beyond `T_max` fails unless you resize weights.

## RoPE (Rotary Position Embedding)

Used in LLaMA, GPT-NeoX, many modern LLMs — **rotate** pairs of dimensions in Q and K by a position-dependent angle. Relative distances encode cleanly; extrapolation tricks (NTK scaling) adjust bases at inference.

## Key takeaways

- Without positional info, attention is **order-blind**.
- **Sinusoidal** = analytic; **learned** = flexible but brittle to longer T.
- **RoPE** is the modern default for autoregressive LMs — learn the API, not every trig identity.
