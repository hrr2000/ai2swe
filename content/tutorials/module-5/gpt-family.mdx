## What “GPT-4.1” style names hide

Each public **model string** (`gpt-4o-mini`, etc.) bundles:

- **Tokenizer + context window**
- **Modalities** (text-only vs vision vs audio)
- **Tool / function calling** format
- **Rate limits, batch APIs, fine-tuning availability**

## Migration checklist

1. **Max tokens / context** — resize prompts.
2. **Stop sequences** — may differ for chat vs completions legacy endpoints.
3. **JSON / schema mode** — verify grammar enforcement level.
4. **Pricing** — per **1M tokens** input vs output often asymmetric.

<Callout type="tip">
Pin **`model=` string** in config, log **prompt token counts**, and feature-flag upgrades — instant rollback beats silent quality drift.
</Callout>

## Key takeaways

- Model IDs are **contracts** — not interchangeable literals.
- Read **deprecation timelines** for hosted APIs the way you read platform EOL notices.
- Maintain an **internal model card** per SKU with eval scores on your tasks.
