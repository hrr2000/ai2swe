## Hardware baseline

- **24 GB** consumer GPU — QLoRA on 7B is realistic.
- **40 GB+** — bf16 LoRA without 4-bit base is smoother.

## Steps

1. **Pick base** — e.g. `mistralai/Mistral-7B-Instruct-v0.3` (verify license).
2. **Format data** — ShareGPT / ChatML JSONL with `messages` array.
3. **Tokenize** with official chat template (`tokenizer.apply_chat_template`).
4. **Attach LoRA** via `PeftModel.from_pretrained` or `LoraConfig` targeting `q_proj`, `k_proj`, `v_proj`, `o_proj`.
5. **Train** with `SFTTrainer` (TRL) — watch **eval loss** + **held-out win-rate**.
6. **Export** — `merge_and_unload()` for single-file deploy or keep adapter weights.

## Sanity checks

- **Generate before/after** on the same prompts — qualitative diff should match intent.
- **Run safety eval** — fine-tunes can **uncage** refusals if dataset biased.

<Callout type="warning">
Never train on **secrets** — adapters memorize API keys seen in fine-tune rows. Scan with `gitleaks` pre-ingest.
</Callout>

## Key takeaways

- **TRL + PEFt** is the fastest path from JSONL to LoRA weights.
- Track **GPU memory**, **tokens/sec**, **eval loss** in one dashboard row per run.
- Decide early whether you **merge** adapters or **hot-load** them in serving.
