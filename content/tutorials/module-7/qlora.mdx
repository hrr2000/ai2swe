## Problem

Even with LoRA, **activations + optimizer states** for a 7B base in fp16 can exceed **24 GB**. **QLoRA** quantizes the **frozen base** aggressively while keeping LoRA paths numerically stable.

## Recipe (conceptual)

1. **4-bit NF4** weights for frozen linear layers — specialized quant format tuned for neural weights.
2. **Double quant** of quantization constants — squeezes extra bits.
3. **Paged AdamW** — spill optimizer state to CPU RAM if VRAM spikes (bitsandbytes feature).

## Caveats

- **Throughput** lower than bf16 full fine-tune — you pay for memory wins with time.
- **Numerical sensitivity** — not every architecture or hyper combo tolerates 4-bit bases; validate loss curves early.

<Callout type="warning">
Merged **4-bit + LoRA** weights are not magically portable across inference engines — verify **vLLM / llama.cpp** support matrices before promising SLAs.
</Callout>

---

## Key takeaways

- QLoRA = **4-bit base + LoRA adapters + memory-aware optimizer**.
- Unlocks **single-GPU** experimentation for many 7B-class models.
- Always **compare evals** against full-precision LoRA baselines — quantization noise matters on tiny datasets.
