## Pipeline stages

1. **Collect** — exports from ticketing, chat logs (with consent), synthetic generations.
2. **Normalize** — strip PII, unify line endings, enforce UTF-8, canonicalize tool names.
3. **Deduplicate** — MinHash near-dup removal prevents memorization spikes.
4. **Stratify splits** — group by user or session so **same author** never crosses train/val.

## Format conventions

Use **JSONL** with explicit keys: `messages`, `tools`, `metadata`. Keep **schema version** field so training code can branch safely.

## Quality metrics

- Labeler **agreement κ**.
- **Toxicity** / policy violation rate (block upstream).
- **Length histograms** — avoid empty completions.

## Key takeaways

- Fine-tune quality ≈ **data QA quality**.
- **Leakage** ruins offline metrics — design splits intentionally.
- Version datasets like **migrations** — reproducible training requires pinned tables.
