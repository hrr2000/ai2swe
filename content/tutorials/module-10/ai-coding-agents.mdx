## Anatomy

- **Planner** proposes file edits + shell commands.
- **Sandbox** runs commands in Docker VMs or WASM microVMs.
- **Verifier** runs `lint`, `typecheck`, `test` after each patch attempt.
- **Human** merges PR after diff review.

## What actually shipped (pattern)

Products converge on **IDE-integrated** agents (Cursor, Copilot Workspace, Jules) with **repo-indexed RAG**, **multi-file diffs**, and **policy hooks** (no `rm -rf /`).

## Risks

- **Secret exfiltration** via prompt injection in dependencies.
- **License contamination** when agents paste unknown snippets.
- **Flaky tests** causing infinite “fix” loops — budget steps.

<Callout type="tip">
Treat agent output like **any contributor PR**: CODEOWNERS, required reviewers, signed commits, SBOM scanning — same supply chain, faster author.
</Callout>

---

## Key takeaways

- Agents = **tools + policy + LLM** in a dev sandbox.
- **CI is the ground truth** — never ship agent patches without automated checks.
- Human **review culture** scales better than chasing fully autonomous merges.
