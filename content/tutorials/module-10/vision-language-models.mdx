## Data flow

Images are resized, patchified, linearly projected, then fed as a **sequence of visual tokens** alongside text tokens. Positional schemes (2D RoPE variants, learned grid embeddings) preserve spatial locality.

## Engineering checklist

- **Resize policy** — extreme aspect ratios distort small text; preserve min side length for OCR.
- **Token budget** — high-res images consume **thousands** of tokens — downscale intentionally.
- **Safety** — faces, IDs, and sensitive screenshots need the same redaction pipeline as text.

## Use cases

- **Receipt / invoice** extraction.
- **UI automation** — locate buttons from screenshots.
- **Visual diffing** — compare diagrams across releases.

<Callout type="tip">
For **charts**, pair VLMs with **structured tool output** (e.g., ask model to emit Vega-Lite JSON) instead of trusting rasterized pixels alone.
</Callout>

## Key takeaways

- VLMs = **shared attention** over fused vision + language tokens.
- Image resolution maps to **cost and latency** — profile before UX promises.
- Treat pixels like **PII** — log policies must redact screenshots.
