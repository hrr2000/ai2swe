## Definition

The **context window** (or **sequence length limit**) is the maximum number of tokens the model architecture and weights were trained to handle in **one** forward pass — prompt + completion (and any hidden scaffolding) included.

Everything beyond that is **truncated**, **summarized**, **retrieved** (RAG), or **windowed** with hacks — the model never “sees” the dropped tokens.

## What really fits in “128k”

Marketing numbers are **upper bounds**, not guarantees of equal quality across all positions. Empirical studies (and many engineers’ logs) show **lost-in-the-middle** effects: models may attend strongly to the **beginning** and **end** of huge prompts while missing details buried mid-document.

<Callout type="warning">
Do not paste 600 pages and assume uniform comprehension. **Structure** matters: headings, bullet lists, repeated task instructions at the end, and smaller well-chosen excerpts beat one giant blob.
</Callout>

## Why windows were small for so long

Vanilla self-attention compares **every token to every other token** — memory and compute scale like **O(T²)** in sequence length (constants matter: hidden size, heads, KV cache). Long-context models need **more VRAM**, **IO bandwidth**, and often **algorithmic tricks** (sparse attention, sliding windows, ring attention, better kernels).

## Engineering strategies

| Situation | Strategy |
| --- | --- |
| Corpus fits comfortably | Put it inline; keep instructions crisp at top and tail. |
| Corpus is huge | **RAG** — retrieve top chunks; only those tokens enter context. |
| Conversation is long | **Summarize** older turns; or drop middle turns; or swap to a long-context model for the rewrite pass. |
| Codebase tasks | Pair long context with **repo map / file list** tools instead of raw dumping when possible. |

## KV cache intuition (preview)

During autoregressive decoding, keys and values from earlier positions can be **cached** so you do not recompute them each new token. Cache size grows with **batch × layers × heads × dim × T** — another reason “more context” hits hardware before it hits marketing slides.

---

## Key takeaways

- Context window = **hard cap** on simultaneous tokens — not “human working memory,” a tensor allocation story.
- Long prompts cost **quadratic attention** in the naive form; **quality may still taper** mid-window.
- For big data, prefer **retrieval + structure** over blind stuffing.
