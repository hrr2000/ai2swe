## Text must become integers

Neural nets run on tensors of numbers. A **tokenizer** is the deterministic program that converts:

```text
raw string → [token_id_1, token_id_2, …, token_id_T]
```

Those integers index **rows of an embedding matrix** (or feed into more complex input pipelines in multimodal models).

## Why not “one token per word”?

Natural language is messy:

- **Out-of-vocabulary** — new slang (“**rizz**”), typos, chemical names.
- **Morphology** — German compounds, Turkish agglutination — word-level vocab explodes.
- **Multilingual** — shared subword units help rare languages.

## Why not “one token per character”?

Sequences get **long**, attention and memory costs grow with length, and models waste capacity learning that `t+h+e` spells “the” from scratch.

## Byte-Pair Encoding (BPE) — the usual answer

**BPE** (and unigram LM variants) start from characters or bytes and **iteratively merge** the most frequent pairs in a corpus. The merges define a vocabulary of substrings:

- Frequent words → **one** token each (`" the "` often compresses well).
- Rare words → **several** subword tokens (`Token` + `ization`).
- Rare characters or emoji → can consume **many** tokens.

<Callout type="tip">
When you estimate cost, measure with the **provider tokenizer** (or `tiktoken`, `sentencepiece`, etc.). Rules of thumb like “4 chars ≈ 1 token” break on code, JSON, Chinese, or emoji-heavy UI copy.
</Callout>

## Billing, limits, and UX

APIs price in **tokens**; context windows cap **T**. That makes tokenization a **product constraint**:

- Pretty-printed JSON is longer than minified JSON — same semantics, higher bill.
- Deep indentation in pasted logs burns whitespace tokens.
- **Same English sentence** can tokenize differently across GPT vs Llama families — do not hard-code offsets from another model’s tokenizer.

## Special tokens

Vocabularies reserve IDs for **control symbols**: start/end of sequence, padding, unknown, user/assistant markers in chat templates. Models only see integers — chat formatting is **tokenizer + template code**, not magic.

---

## Key takeaways

- Tokenizers are **versioned programs** paired with checkpoints — always ship them together.
- **Subword** methods balance vocabulary size, OOV robustness, and sequence length.
- Token counts drive **money, latency, and whether your prompt fits** — profile them like payload size.
