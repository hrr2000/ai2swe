## Why not `===` for text?

Exact string match cannot find “**automobile**” when the user typed “**car**.” Embeddings lift text into ℝⁿ so **nearby points ≈ similar meaning**. A retriever’s job is: given query vector `q`, find database vectors `{d_i}` with smallest distance / largest similarity.

## Cosine similarity — direction matters

```text
cos_sim(a, b) = (a · b) / (||a|| ||b||)
```

- **1** — same direction (often “same topic” for normalized sentence embeddings).
- **0** — orthogonal (unrelated in first-order geometry).
- **-1** — opposite direction (rare for text, but possible).

Cosine **ignores magnitude** after normalization — useful when short queries match long documents that point the same way in embedding space.

## Euclidean (L2) distance — “as the crow flies”

```text
d₂(a, b) = ||a − b||₂
```

Magnitude matters: doubling a vector can move it far in L2 even if direction is unchanged. Vision models sometimes use L2 on **unnormalized** features; text pipelines often **L2-normalize** first so cosine and squared L2 rankings align.

## Dot product — when lengths encode importance

If you **do not** normalize, a longer vector can have a larger inner product simply because it is bigger. Some systems intentionally use **dot product attention** or unnormalized scores — just know what geometry you are buying.

<Callout type="tip">
Start with **cosine** for semantic text retrieval with sentence embeddings. If results look off, verify **normalization**, **embedding model choice**, and **chunking** before you blame the metric.
</Callout>

## Brute force vs ANN

Exact search is **O(N)** per query — fine for thousands of rows, painful for tens of millions. **Approximate nearest neighbors (ANN)** — HNSW, IVF, LSH families — precompute a graph or partition structure so each query touches only a **small neighborhood** of candidates.

Recall@k may dip slightly vs brute force; latency and RAM win orders of magnitude.

---

## Key takeaways

- **Cosine** ≈ directional match; **L2** ≈ geometric distance; **dot product** mixes direction and length.
- Normalize vectors when you want cosine-like behavior from inner products.
- Production search uses **ANN indexes** (pgvector IVFFlat/HNSW, Pinecone, Milvus, …) under the hood.
