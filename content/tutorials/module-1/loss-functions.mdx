## From boolean tests to differentiable scores

Unit tests assert discrete pass/fail. Models output **scores over many possible answers** (logits, bounding boxes, token distributions). A **loss function** `L(y_hat, y)` measures how far predictions are from targets and is chosen so it is **smooth enough** to differentiate almost everywhere — that is what makes gradient-based training possible.

  code="expect(actual).toBe(expected) — binary signal"
  ai="loss = f(prediction, label) — scalar signal with slope w.r.t. every weight"

## Regression: mean squared error (MSE)

For real-valued targets (house prices, coordinates):

```text
MSE = (1/N) Σ || y_hat_i − y_i ||²
```

Squaring penalizes large errors aggressively. Gradients are linear in the residual — easy optimization geometry for many convex-ish problems.

## Classification: cross-entropy with logits

When labels are categories, models emit **logits** `z` (unnormalized scores). **Softmax** turns logits into probabilities `p`, and **cross-entropy** compares `p` to the one-hot label `y`:

```text
CE(y, p) = − Σ_k y_k log p_k
```

In libraries you often see `cross_entropy_with_logits` — it fuses softmax + CE in a **numerically stable** way. This is the default head for multi-class image classifiers and, at each timestep, for language modeling over the vocabulary.

<Callout type="insight">
Cross-entropy is not arbitrary — it is the negative log-likelihood under a **multinomial model**. You are doing maximum likelihood estimation with gradient descent.
</Callout>

## Why not only accuracy?

Accuracy is a **step function** — tiny weight changes rarely flip a discrete argmax, so gradients are zero almost everywhere. Loss functions provide a **graded** signal: “you were wrong, but you were closer to the right class than last step.”

## Auxiliary and composite losses

Real systems stack objectives: detection adds bbox + classification terms; diffusion adds noise-prediction terms; RLHF adds preference losses. Each term should be **scaled** so one component does not drown the others — the same judgment you use balancing metrics in a product scorecard.

---

## Key takeaways

- **Loss** turns “how wrong?” into a **scalar** suitable for optimization.
- **MSE** for regression; **cross-entropy** for classification / token prediction.
- Training needs **differentiable** surrogates; **accuracy/F1** remain what you report to humans.
