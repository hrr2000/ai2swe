## From one neuron to a stack

A single neuron draws one linear decision boundary (then bends it slightly with an activation). Real problems need **many** boundaries composed together. We organize neurons into **layers** so we can reason about shape changes and reuse fast linear algebra kernels.

## The three names every diagram uses

1. **Input layer** — numbers you provide: pixels flattened to a vector, token IDs embedded into vectors, tabular features, etc.
2. **Hidden layers** — everything between input and output. “Hidden” simply means you do not supervise their values directly; you only supervise the final head.
3. **Output layer** — produces task-specific logits or scalars: ten numbers for MNIST classes, a single sigmoid for “spam probability,” a sequence of logits over the vocabulary for language models.

Each hidden layer typically does:

```text
x_{l+1} = activation( W_l @ x_l + b_l )
```

Same pattern as a neuron block, but `W_l` is a **matrix** so many neurons run at once — that is how width and depth appear in code.

## “Deep” means depth, not mysticism

**Deep learning** = **many** hidden layers chained. Depth buys **hierarchy**: early vision layers detect edges and textures; deeper layers compose parts into objects. Depth without meaningful structure or data is just an expensive identity attempt — engineering still matters.

## Shapes: the discipline that keeps you sane

Frameworks track **tensor shapes** the way TypeScript tracks types:

```text
[batch, 784] → Linear(784→256) → [batch, 256] → ReLU → [batch, 256] → … → [batch, 10]
```

When training fails mysteriously, the first printouts are often **shape mismatches** — the same class of bug as passing the wrong DTO between services.

## Why layer boundaries are useful

Layers bundle parameters and make **checkpointing** and **transfer learning** practical: you can freeze early layers (general features), tune later layers (task-specific), or graft a new output head for a different label set.

<Callout type="tip">
When reading a paper or `config.json`, map each block (`Linear`, `Conv2d`, `LayerNorm`, `MultiHeadAttention`) to “what tensor shape goes in and what comes out.” That map is the real architecture diagram.
</Callout>

## Key takeaways

- Networks are **layered function composition**; each layer is a parameterized transform plus (usually) a nonlinearity.
- **Depth** stacks abstractions; **width** increases parallel decisions per layer.
- Think in **tensor shapes** — it is the static typing of neural nets.
