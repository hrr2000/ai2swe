## Why linear layers are not enough

Two linear maps composed are still linear: `W2 @ (W1 @ x) = (W2 @ W1) @ x`. A tall stack of “matrix multiply only” would be equivalent to **one** matrix — no extra expressive power.

**Activation functions** break linearity. They act element-wise (mostly) and let the network carve curved decision regions instead of a single hyperplane sandwich.

  ai="Activations: steep but smooth ramps the optimizer can walk down with gradients"

## ReLU — the workhorse

**Rectified Linear Unit** — keep positive values, zero out negatives:

```javascript
function relu(x) {
  return Math.max(0, x);
}
```

Properties engineers care about:

- **Cheap** — branchless on many accelerators; sparse activations when inputs are negative.
- **Stable gradients** for positives (gradient 1), zeros for negatives (neurons can “die” if never activated — variants like LeakyReLU exist).

## Sigmoid and tanh — bounded outputs

**Sigmoid** maps ℝ → (0, 1) — interpretable as a probability when the problem is binary:

```javascript
function sigmoid(x) {
  return 1 / (1 + Math.exp(-x));
}
```

**Tanh** maps ℝ → (-1, 1) — zero-centered, once popular in RNNs; still appears inside some gates.

<Callout type="insight">
Use **sigmoid** on a **single logit** for “probability of class A.” For **multi-class**, prefer **softmax** on the output vector — it generalizes sigmoid so classes compete for probability mass.
</Callout>

## GELU, SiLU, and friends in modern stacks

Transformers often use **GELU** or **SiLU (Swish)** — smooth, self-gated curves that empirically work well with AdamW and deep stacks. You rarely hand-implement them; treat them as “better defaults” when building on pretrained checkpoints.

## Failure mode: vanishing gradients (historical context)

Deep **sigmoid/tanh** chains squash inputs into flat regions where gradients are tiny — optimization crawls. ReLU family + residual connections (later courses) largely fixed day-to-day training for vision and language backbones.

---

## Key takeaways

- Without nonlinearities, depth buys you nothing — **activations** are what make deep nets expressive.
- **ReLU** is the default hidden activation for many feedforward and conv models.
- **Sigmoid/softmax** still matter on **outputs** where you need calibrated probabilities.
