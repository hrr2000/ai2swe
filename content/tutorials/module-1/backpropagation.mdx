## The problem it solves

After a forward pass you know the **loss**. You want **partial derivatives** of that loss with respect to **every parameter** — millions or billions of them. Writing those formulas by hand is impossible. **Reverse-mode automatic differentiation** (popularly called **backpropagation** in neural nets) automates it.

## Graph picture

Build a **directed acyclic graph** while executing the forward pass:

- **Leaves** — constants, parameters `W`, inputs `x`.
- **Interior nodes** — `matmul`, `relu`, `softmax`, etc.

Each node knows how to compute **local gradients** (how its outputs wiggle when its inputs wiggle). The chain rule stitches those into **global** `∂loss/∂tensor`.

## Reverse pass intuition

1. Seed `∂loss/∂loss = 1`.
2. Visit nodes from output toward inputs (reverse topological order).
3. For each node, apply the **chain rule** to propagate gradients to parents.

Matmul example sketch: if `Y = XW`, then `∂L/∂X = (∂L/∂Y) @ Wᵀ` and `∂L/∂W = Xᵀ @ (∂L/∂Y)` — structure matters less than the takeaway: **linear ops route gradients** similarly to how they route activations.

## Memory trade-off

Reverse-mode AD stores activations from the forward pass (or recomputes with checkpointing) because backward formulas need them. Big models + long sequences ⇒ **VRAM for training** ≫ inference — another reason training and serving are different jobs.

<Callout type="tip">
Frameworks hide backprop behind `.backward()`. Your job is to choose **losses**, **architectures**, and **batch shapes** that keep gradients **finite** and **non-vanishing** — monitoring gradient norms is normal in serious training runs.
</Callout>

## Backprop does not update weights

It **computes** `∇_θ L`. The **optimizer** (SGD, Adam, …) actually applies updates — covered next with **gradient descent**.

---

## Key takeaways

- Backprop = **reverse-mode AD** through the computation graph.
- It yields **∂loss/∂parameter** for all parameters in one backward sweep.
- Implementation is a solved problem in PyTorch/JAX/TensorFlow — conceptual understanding helps you debug **exploding/vanishing** gradients and shape errors.
