## Forget Biology. Think Functions.

The word "neuron" makes everyone picture brain cells and biology. That's actively unhelpful for understanding how neural networks work.

Here's a better framing: **a neuron is just a function**.

```typescript
function neuron(inputs: number[], weights: number[], bias: number): number {
  // Step 1: Multiply each input by its weight and sum them up
  const weightedSum = inputs.reduce((sum, x, i) => sum + x * weights[i], 0);

  // Step 2: Add a bias (like a default offset)
  const z = weightedSum + bias;

  // Step 3: Pass through an activation function (more on this below)
  return relu(z);
}

function relu(x: number): number {
  return Math.max(0, x);  // That's the entire ReLU activation function
}
```

That's a neuron. **Three steps**. Weighted sum → add bias → activate.

## The Weighted Sum: Making a Decision

Imagine you're deciding whether to deploy code on a Friday. You consider several signals:

```typescript
const signals = {
  testsGreen:      1.0,  // Tests passed? (1 = yes, 0 = no)
  teamAvailable:   0.5,  // Is the team online? (partial)
  featureUrgent:   0.8,  // How urgent is the feature?
  recentIncident:  0.0,  // Recent incident? (0 = no incident)
};

// These are your learned "weights" — how much each signal matters
const weights = {
  testsGreen:     3.5,   // Tests matter a lot
  teamAvailable:  2.0,   // Team availability matters
  featureUrgent:  1.5,   // Urgency matters somewhat
  recentIncident: -4.0,  // Recent incident is a strong negative
};

const score =
  signals.testsGreen     * weights.testsGreen     +  // 1.0 * 3.5 = 3.5
  signals.teamAvailable  * weights.teamAvailable  +  // 0.5 * 2.0 = 1.0
  signals.featureUrgent  * weights.featureUrgent  +  // 0.8 * 1.5 = 1.2
  signals.recentIncident * weights.recentIncident;   // 0.0 * -4.0 = 0.0

console.log(score); // → 5.7 (deploy?)
```

A neuron does exactly this. It takes *n* inputs, multiplies each by a learned weight, and sums the results. The **weights are what gets learned during training**.

## The Bias: The Default Offset

A bias lets the neuron "fire" even when all inputs are zero. Think of it like a baseline:

```typescript
// Without bias: if all inputs are 0, output is always 0
const z = weightedSum;  // 0

// With bias: the neuron has a "default tendency"
const bias = -1.5;  // This neuron is a bit reluctant to activate
const z = weightedSum + bias;  // Even if signals are middling, the bar is high
```

In code terms: bias is like a default function argument that shifts the decision threshold.

<Callout type="insight">
**Key Insight:** `weights` and `bias` are the "parameters" of the neuron. During training, the optimizer adjusts them to make the neuron better at its job — just like you'd tune constants in a configuration file.
</Callout>

## The Activation Function: The Soft If/Else

After the weighted sum, we pass the result through an **activation function**. Why? Because if we didn't, every layer would just be linear math — and you can collapse any number of linear operations into a single one. Activation functions break that linearity.

Think of them as soft, differentiable versions of `if/else`:

```typescript
// ReLU — the most common activation
// "If input > 0, pass it through. Otherwise, output 0."
function relu(x: number): number {
  return Math.max(0, x);
  // Equivalent if/else: if (x > 0) return x; else return 0;
}

// Sigmoid — squishes output to [0, 1] range
// Great for "probability" style outputs
function sigmoid(x: number): number {
  return 1 / (1 + Math.exp(-x));
  // Think of: probability this condition is true
}

// Tanh — squishes to [-1, 1]
// Used when negative values are meaningful
function tanh(x: number): number {
  return Math.tanh(x);
}
```

**ReLU** is the most used in modern networks. It's literally just `Math.max(0, x)`.

## Putting It All Together

Here's a complete neuron, with real numbers:

```python
import numpy as np

def neuron(inputs, weights, bias, activation="relu"):
    # Step 1: Weighted sum
    z = np.dot(inputs, weights) + bias
    # np.dot([1, 2, 3], [0.5, -0.3, 0.8]) = (1*0.5) + (2*-0.3) + (3*0.8) = 2.3

    # Step 2: Activation
    if activation == "relu":
        return max(0, z)
    elif activation == "sigmoid":
        return 1 / (1 + np.exp(-z))
    else:
        return z  # Linear (no activation)

# Example
inputs  = [0.8, 0.2, 0.5]    # Three input features
weights = [1.2, -0.5, 0.9]   # Learned weights
bias    = -0.3                # Learned bias

output = neuron(inputs, weights, bias)
print(output)
# z = (0.8*1.2) + (0.2*-0.5) + (0.5*0.9) + (-0.3) = 0.96 - 0.1 + 0.45 - 0.3 = 1.01
# relu(1.01) = 1.01
```

## Why One Neuron Isn't Enough

A single neuron can only learn one linear boundary — like a single `if` statement. That's why we stack them.

Next lesson, you'll see how chaining neurons into **layers** (like middleware) creates the ability to learn complex patterns — even image classification, language understanding, and coding.

<Callout type="tip">
**Quick test:** Look at the `relu` function: `Math.max(0, x)`. That's literally the most important activation function in deep learning. If you understand it, you understand 80% of modern AI activations.
</Callout>

## Try It Yourself

Open your terminal and run this quick experiment:

```python
# See how a neuron responds to different inputs
import numpy as np

def neuron(x, w, b):
    return max(0, np.dot(x, w) + b)

# Imagine this detects "is this sentence positive?"
weights = [2.0, -1.5, 1.8]  # weights for: exclamation, negativity, enthusiasm
bias = -0.5

test_cases = [
    ([1, 0, 1], "excited positive: '!'=1, neg=0, enthus=1"),
    ([0, 1, 0], "negative: '!'=0, neg=1, enthus=0"),
    ([0, 0, 0], "neutral: all zeros"),
]

for inputs, label in test_cases:
    output = neuron(inputs, weights, bias)
    print(f"{label} → {output:.2f}")
```

**Expected output:**
```
excited positive: '!'=1, neg=0, enthus=1 → 3.30
negative: '!'=0, neg=1, enthus=0 → 0.00
neutral: all zeros → 0.00
```

Notice how the negative test fires `0`. That's ReLU suppressing negative signals.

## Key Takeaways

- A neuron is a **function**: `f(inputs, weights, bias) → number`
- **Weights** = how much each input matters (learned during training)
- **Bias** = the neuron's "default offset" (also learned)
- **Activation functions** = the soft if/else that adds non-linearity (ReLU = `Math.max(0, x)`)
- Single neuron = one linear decision. Stack them → complex decisions
