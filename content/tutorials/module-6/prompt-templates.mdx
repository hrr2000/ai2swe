## Anti-pattern

```typescript
const prompt = "Summarize: " + userText; // injection + formatting chaos
```

## Pattern

```typescript
const summarizeTpl = (doc: string, bullets: number) => `
You are a concise analyst.
Return exactly ${bullets} bullet points for:
<<<DOC
${doc}
DOC
`.trim();
```

Use **real template engines** (Mustache, Handlebars, lit-html) when localization teams get involved.

## Safety

- **Delimit** untrusted text with clear markers (`<<<DOC`) so the model knows boundary.
- **Escape** XML/JSON if downstream expects structured wrappers.
- **Redact** PII before inserting into templates — logs will contain expanded prompts.

<Callout type="insight">
Store templates next to **eval fixtures** in git. PR review on prompt diffs catches accidental tone or schema regressions.
</Callout>

## Key takeaways

- Templates = **reuse + reviewability** for prompts.
- Always separate **instruction skeleton** from **user payload**.
- Treat prompt repos like **frontend strings** — lint, snapshot-test, translate.
