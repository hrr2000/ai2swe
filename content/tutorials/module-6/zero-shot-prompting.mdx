## Zero-shot

Only natural-language instructions:

```text
Translate JSON keys from camelCase to snake_case. Return JSON only.
{ "userName": "Ada" }
```

Works when the task is **common in pretraining** and your instructions are crisp.

## Few-shot (in-context learning)

Add **2–8 demonstrations** — pairs of `(input, ideal output)` before the real input:

```text
Q: hi → A: bonjour
Q: thanks → A: merci
Q: see you → A:
```

The model continues the pattern. You are not updating weights; you are **steering attention** with exemplars.

## Tradeoffs

| Few-shot | Cost |
| --- | --- |
| Better format adherence | More **tokens** + latency |
| Stabilizes edge formats | Risk of **accidental bias** from chosen examples |

<Callout type="tip">
Curate few-shots like **testdata/** in a repo — version them, diversify edge cases, and never let them contain **secrets** (PII, keys) because they echo in logs.
</Callout>

---

## Key takeaways

- **Zero-shot** for simple, well-pretrained tasks.
- **Few-shot** when format is exotic or metrics jump measurably.
- Demonstrations are **prompt tokens**, not training — budget them.
