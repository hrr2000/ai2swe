## Pipeline

1. **Index** — walk repo respecting `.gitignore`, hash files, skip binaries.
2. **Chunk** — prefer **function-level** splits with signature in chunk header.
3. **Embed** — code embedding model (same for query + doc).
4. **Retrieve** — top-k chunks + optional **BM25 hybrid**.
5. **Generate** — system prompt mandates **file:line** citations; stream SSE.

## UX details

- Show **retrieved snippets** in UI disclosure before model paraphrases.
- **Re-rank** with cross-encoder if quality insufficient.
- **Refresh index** on git commit webhook.

<Callout type="insight">
For **monorepos**, namespace chunks by package + owner team metadata so retrieval filters reduce cross-team noise.
</Callout>

## Key takeaways

- Code RAG wins when **chunk boundaries respect syntax**, not naive token cuts.
- **Citations** are non-negotiable for developer trust.
- Automate **index rebuilds** like nightly CI artifacts.
