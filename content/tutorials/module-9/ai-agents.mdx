## Minimal agent architecture

- **Planner** — LLM proposes next tool or final answer.
- **Executor** — runs tool with sandboxing.
- **Memory** — scratchpad + retrieval over past steps.
- **Stop policy** — max steps, max cost, deadline, or explicit `done` tool.

## Failure classes

- **Infinite loops** — model alternates between two useless tools.
- **Partial observability** — missing metrics cause wrong tool picks.
- **Reward hacking** — model satisfies literal metric while violating intent.

## Mitigations

- **Step budgets** + exponential backoff on errors.
- **Second-opinion model** for high-stakes tool args.
- **Deterministic verifiers** (unit tests) after code-edit tools.

<Callout type="insight">
Ship **read-only** agents first (search, summarize). **Write** paths (tickets, deploys) need the same change-management rigor as human operators.
</Callout>

## Key takeaways

- Agents = **LLM + control loop + tools** — not a new fundamental model.
- **Budgets and verifiers** matter more than clever prompts alone.
- Design for **abort, resume, audit** from day one.
