## Goals

Each chunk should be **semantically self-contained** enough to answer a user question without needing distant paragraphs — yet small enough to fit **context + leave room** for the answer.

## Fixed-size windows

Split every `N` tokens with `overlap` tokens repeated — trivial, fast, terrible across **Markdown headings**, **JSON**, or **legal clause** boundaries.

## Structure-aware splits

- Respect **Markdown `#` headers** — keep sections together until max length exceeded.
- **Code** — split on function/class boundaries using tree-sitter.
- **Tables** — never slice mid-row if numeric answers depend on headers.

## Metadata

Attach `source_path`, `heading`, `commit_sha`, `page` to each chunk for **citations** and debugging.

<Callout type="tip">
Run **k-fold eval** over chunk sizes on a labeled query set — the winning `N` is empirical, not universal.
</Callout>

## Key takeaways

- Chunking quality caps **RAG ceiling** — invest before buying bigger models.
- **Overlap** reduces boundary misses but increases index size — tune deliberately.
- Pair text chunks with **rich metadata** for filters (`tenant_id`, `language`).
