## Defense in depth

1. **Transport auth** — OAuth, mTLS, org scoping.
2. **Input hygiene** — max length, Unicode normalization, profanity filters where legally required.
3. **Prompt injection detectors** — separate lightweight model or rules spotting `ignore previous instructions`.
4. **Output filters** — PII redaction, code static analysis before returning to user.
5. **Human escalation** queue for edge cases.

## Jailbreak reality

No filter is perfect. Combine **system policies**, **tool allow-lists**, and **monitoring** instead of promising “100% safe AI.”

<Callout type="warning">
Never rely on **another LLM** as your sole security boundary unless its false positive rate is acceptable for revenue — dual-model costs add up.
</Callout>

## Key takeaways

- Guardrails are **product + security engineering**, not a checkbox.
- **Log near-misses** to refine rules without exposing user content broadly.
- Pair technical filters with **usage policies** and **legal review**.
